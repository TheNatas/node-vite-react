### Objetivo: Criar uma aplicação de lista de tarefas

### Requisitos:

  - CRUD completo de tarefas
  - Filtros (todas, pendentes, concluídas)
  - Busca por título/descrição
  - Persistência local (LevelDB/SQLite)
  - Interface responsiva
  - Validação de formulários

### Tech Stack:

  - Frontend: Vite + React + Bootstrap
  - Backend: Node.js 22
  - Code: TypeScript

### Diferencial:

  - Teste frontend e backend
  - Importar/Exportar dados
  - Temas claro/escuro
  - Cadastro/Login de usuários

### Critérios de Avaliação (Por order de prioridade)

  - ✅ Qualidade do código
  - ✅ Qualidade dos Testes
  - ✅ Organização da estrutura
  - ✅ Tratamento de erros
  - ✅ UX/UI e responsividade
  - ✅ Documentação (README)

### Dicas

  - Uso é permitido de ferramentas de IA, templates, frameworks, etc.
  - Tudo é permitido, faça seu melhor no tempo estipulado.
  - Faça primeiro um deploy com um "hello world", para testar e ver se seu projeto está rodando normalmente.
  - Projetos que não rodarem estão desclassificados.
  - Fazer o Deploy é um ponto extra, mas não é obrigatório.

### Instruções

  - Leia atentamente o desafio acima
  - Implemente a solução no repositório GitHub que você informou
  - Faça commits regulares conforme desenvolve
  - Certifique-se de completar antes do tempo acabar - commits após o deadline não serão aceitos
  - A aplicação deve rodar na porta 3000
